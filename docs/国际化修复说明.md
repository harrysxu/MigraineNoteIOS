# 国际化修复说明

## 问题诊断

### 根本原因

项目的国际化配置存在不匹配问题：

- **Xcode 项目设置**: `developmentRegion = en` （Development Language = 英文）
- **Localizable.xcstrings**: `sourceLanguage = "zh-Hans"` （源语言 = 简体中文）

这种不匹配导致系统在选择语言资源时出现混乱。当设备语言设置为英文时，应用会回退到源语言（简体中文），因此即使系统语言是英文，界面仍然显示中文。

### 技术背景

在 iOS 国际化系统中：
1. **Development Region** 定义项目的基础语言（在 `project.pbxproj` 中）
2. **Source Language** 定义 String Catalog 的源语言（在 `.xcstrings` 文件中）
3. 这两个设置**必须一致**，否则会导致语言回退机制失效

## 修复方案

### 已完成的修改

1. **修改 `Localizable.xcstrings` 的源语言**

```diff
{
-  "sourceLanguage" : "zh-Hans",
+  "sourceLanguage" : "en",
  "strings" : {
```

2. **项目配置保持不变**（已经是正确的）

```
developmentRegion = en;
knownRegions = (
  en,
  Base,
  "zh-Hans",
  "zh-Hant",
);
```

### 语言优先级逻辑

修复后的语言选择流程：

```
设备语言: 英文 (en)
→ 匹配 knownRegions: ✓ en
→ 使用 Localizable.xcstrings 中的 "en" 翻译
→ 界面显示: 英文 ✓

设备语言: 简体中文 (zh-Hans)
→ 匹配 knownRegions: ✓ zh-Hans
→ 使用 Localizable.xcstrings 中的 "zh-Hans" 翻译
→ 界面显示: 简体中文 ✓

设备语言: 繁体中文 (zh-Hant)
→ 匹配 knownRegions: ✓ zh-Hant
→ 使用 Localizable.xcstrings 中的 "zh-Hant" 翻译
→ 界面显示: 繁体中文 ✓
```

## 注意事项

### 1. 标签初始化策略

根据 `docs/国际化简化说明.md`，应用采用**一次性初始化策略**：

- 标签（症状、诱因等）在应用**首次启动时**根据系统语言初始化
- 后续切换系统语言**不会**重新初始化标签
- 这是设计决策（YAGNI 原则），因为 < 1% 的用户需要动态语言切换

### 2. 不影响的内容

以下内容**不会**随语言切换：

- 用户自定义的标签名称
- 已保存的记录中的 `displayName` 字段
- 用户输入的备注和笔记

### 3. 清理应用数据测试

如果需要测试标签初始化，需要：

```bash
# 删除应用数据（模拟器）
xcrun simctl uninstall booted com.yourcompany.migraine_note

# 重新安装
# 在 Xcode 中 Cmd+R 运行
```

## 总结

通过将 `Localizable.xcstrings` 的 `sourceLanguage` 修改为 `en`，使其与 Xcode 项目的 `developmentRegion` 保持一致，修复了国际化不生效的问题。

现在应用能够正确响应系统语言设置，在英文、简体中文、繁体中文环境下都能正确显示相应语言的界面文本。
