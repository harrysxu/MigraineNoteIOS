## 数据兼容性

### 完全向后兼容 ✅

1. **现有记录不受影响**
   - `AttackRecord` 存储的是 `displayName` 字符串（如"恶心"），与 `CustomLabelConfig` 解耦
   - 切换语言后，已有记录仍显示原始语言的 displayName
   
2. **LabelManager 策略**
   - 默认标签初始化时使用当前系统语言的本地化 displayName
   - 用户自定义标签的 displayName 保持原样
   - labelKey 保持英文键，便于跨语言识别

3. **iCloud 同步兼容**
   - labelKey 使用英文键，多设备间同步不会出现语言混淆
   - 去重机制基于 (category, labelKey, subcategory)，语言无关

## 技术亮点

1. **使用 Swift String Catalog (.xcstrings)**
   - 统一管理所有翻译
   - 支持自动生成 Swift 符号
   - 易于维护和扩展

2. **枚举本地化模式**
   - rawValue 使用英文键（便于编程和存储）
   - localizedName 计算属性提供本地化显示
   - 数据层与展示层完全分离

3. **动态标签本地化**
   - LabelManager 自动根据当前语言生成 displayName
   - 无需硬编码，易于添加新语言

4. **InfoPlist.strings**
   - 权限说明支持三语言
   - 应用名称支持本地化

## 使用方法

### 开发者

#### 1. 在代码中使用本地化字符串

```swift
// UI 文本
Text("common.save")  // 自动本地化

// 枚举显示
let category = MedicationCategory.nsaid
Text(category.localizedName)  // 显示"非甾体抗炎药(NSAID)" / "非甾體抗炎藥(NSAID)" / "NSAIDs"
```

#### 2. 添加新的翻译

编辑 `Localizable.xcstrings`，添加新键及其三种语言翻译：

```json
"new.key" : {
  "localizations" : {
    "en" : { "stringUnit" : { "state" : "translated", "value" : "New Text" } },
    "zh-Hans" : { "stringUnit" : { "state" : "translated", "value" : "新文本" } },
    "zh-Hant" : { "stringUnit" : { "state" : "translated", "value" : "新文本" } }
  }
}
```

#### 3. 添加新的枚举本地化

```swift
enum NewEnum: String, CaseIterable {
    case value1 = "value1"
    
    var localizedName: String {
        String(localized: String.LocalizationValue("new.enum.\(rawValue)"))
    }
}
```

然后在 `Localizable.xcstrings` 中添加对应翻译。

### 用户

1. **切换语言**
   - 进入 iOS 设置 > 通用 > 语言与地区
   - 选择 简体中文 / 繁體中文 / English
   - 重启应用即可

2. **查看效果**
   - 所有界面文字自动切换
   - 权限弹窗显示对应语言
   - 应用名称显示对应语言
