# 国际化快速参考指南

## 如何使用本地化

### 1. 在 SwiftUI 视图中使用

```swift
// 简单文本
Text("common.save")  // 自动本地化为 "保存" / "儲存" / "Save"

// 带变量的文本（需要在 Localizable.xcstrings 中定义）
Text("stats.days.count", comment: "%d 天")
```

### 2. 使用枚举的本地化名称

```swift
// 所有枚举都有 localizedName 属性
let category = MedicationCategory.nsaid
Text(category.localizedName)  // 显示 "非甾体抗炎药(NSAID)"

// 示例
ForEach(MedicationCategory.allCases) { category in
    Text(category.localizedName)
}
```

## 添加新的翻译

### 方法 1: 直接编辑 Localizable.xcstrings

```json
{
  "new.key" : {
    "comment" : "描述这个键的用途",
    "extractionState" : "manual",
    "localizations" : {
      "en" : {
        "stringUnit" : {
          "state" : "translated",
          "value" : "English Text"
        }
      },
      "zh-Hans" : {
        "stringUnit" : {
          "state" : "translated",
          "value" : "简体中文"
        }
      },
      "zh-Hant" : {
        "stringUnit" : {
          "state" : "translated",
          "value" : "繁體中文"
        }
      }
    }
  }
}
```

### 方法 2: 使用 Xcode 界面

1. 打开 `Localizable.xcstrings`
2. 点击 `+` 按钮添加新键
3. 为每种语言填写翻译

## 测试不同语言

### 在模拟器中切换语言

1. 打开 iOS 设置（Settings）
2. 选择 General（通用） > Language & Region（语言与地区）
3. 添加或切换到需要的语言
4. 重启应用查看效果

### 在 Xcode 中预览不同语言

在 SwiftUI Preview 中：

```swift
#Preview("简体中文") {
    ContentView()
        .environment(\.locale, .init(identifier: "zh-Hans"))
}

#Preview("繁体中文") {
    ContentView()
        .environment(\.locale, .init(identifier: "zh-Hant"))
}

#Preview("English") {
    ContentView()
        .environment(\.locale, .init(identifier: "en"))
}
```

## 常见问题

### Q: 为什么我的翻译没有生效？

1. 确保 Localizable.xcstrings 中有对应的键
2. 确保使用的是正确的键名（区分大小写）
3. Clean Build Folder (Cmd+Shift+K) 然后重新编译
4. 重启应用或模拟器

### Q: 如何为 Info.plist 添加翻译？

编辑对应语言的 `InfoPlist.strings` 文件：
- `zh-Hans.lproj/InfoPlist.strings` - 简体中文
- `zh-Hant.lproj/InfoPlist.strings` - 繁体中文
- `en.lproj/InfoPlist.strings` - 英文

### Q: 枚举的 rawValue 为什么要改成英文？

1. **数据存储**：英文 rawValue 在数据库中更稳定，不受语言变化影响
2. **代码可读性**：英文键更易于理解和维护
3. **跨平台**：如果将来需要 Android 版本，英文键可以共用
4. **调试**：英文键在日志和调试中更清晰

### Q: 已有数据会受影响吗？

不会。因为：
1. `AttackRecord` 存储的是 `displayName` 字符串（如"恶心"），不是枚举 rawValue
2. 已有记录会保持原始语言
3. 新记录会使用当前系统语言的翻译
