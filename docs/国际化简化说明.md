# 国际化简化说明

## 实施日期
2026年2月24日

## 概述
基于实际使用场景分析，我们简化了国际化方案，去除了动态语言切换支持，采用首次启动时一次性初始化的方式。

## 核心设计理念

### YAGNI 原则（You Aren't Gonna Need It）
- 不为极少数边缘场景（< 1%）增加不必要的复杂度
- 关注实际用户需求：个人健康管理App的用户极少切换系统语言
- 保持代码简单清晰，降低维护成本

## 架构设计

### 标签初始化策略

**一次性初始化**：App在每个设备首次启动时，根据当前系统语言初始化所有默认标签。

```swift
// LabelManager.swift
func insertIfNotExists(...) {
    // displayName 在首次创建时根据当前系统语言一次性初始化
    let finalDisplayName: String
    if isDefault && !labelKey.isEmpty {
        let locKey = "label.\(category).\(labelKey)"
        finalDisplayName = String(localized: String.LocalizationValue(locKey))
    } else {
        finalDisplayName = displayName
    }
}
```

**初始化示例**：
- 中文系统首次启动：创建标签 `{labelKey: "nausea", displayName: "恶心"}`
- 英文系统首次启动：创建标签 `{labelKey: "nausea", displayName: "Nausea"}`

### 数据模型

```swift
@Model
final class CustomLabelConfig {
    var labelKey: String = ""      // 唯一标识符（英文，用于去重）
    var displayName: String = ""   // 显示名称（一次性初始化）
    var isDefault: Bool = false    // 是否为默认标签
    
    // 注意：删除了 localizedDisplayName 计算属性
    // displayName 直接用于显示，不再动态翻译
}
```

### iCloud 同步策略

**去重规则**：保持基于 `labelKey` 的去重，避免语义重复的标签。

```swift
// 去重键：(category, labelKey, subcategory)
// 保留最早创建的记录（createdAt 最小）
```

**同步场景示例**：
```
设备 A（中文）：{labelKey: "nausea", displayName: "恶心"}
设备 B（英文）：{labelKey: "nausea", displayName: "Nausea"}

iCloud 同步后，去重保留设备 A 的记录：
两个设备都显示：{labelKey: "nausea", displayName: "恶心"}

影响：设备 B 用户会看到中文"恶心"而不是英文"Nausea"
权衡：这种情况极少发生（多设备不同语言 < 5%），用户可手动重命名
```

## 用户体验影响

### ✅ 绝大多数用户（> 99%）

**无任何影响**：
- 首次使用看到正确语言的标签
- 记录、查看、导出功能完全正常
- 性能略有提升（无需动态翻译）

### ⚠️ 少数边缘场景

#### 场景1：用户切换系统语言（< 1%）
```
初始状态（中文系统）：
- 标签："恶心"、"搏动性"

用户切换到英文系统：
- UI 界面：英文 ✅
- 标签仍然：中文 ⚠️

解决方案：
- 用户可在标签管理中重命名
- 或者通过App设置重新初始化标签
```

#### 场景2：多设备不同语言（< 5%）
```
设备 A（中文）+ 设备 B（英文）：
- 可能存在重复标签（"恶心" 和 "Nausea"）
- 或者一个设备显示另一语言的标签

解决方案：
- 用户可隐藏/删除不需要的标签
- 或者在一个设备上重命名标签
```

## 系统界面多语言支持

### ✅ 完整支持的内容

以下内容**完全支持**动态多语言切换：

1. **UI 文本**：所有界面文本从 `Localizable.xcstrings` 读取
2. **枚举本地化**：所有枚举的 `localizedName` 动态翻译
3. **权限说明**：`InfoPlist.strings` 支持三种语言

**示例**：
```swift
// 枚举动态翻译（完全支持）
enum PainLocation {
    case forehead
    
    var localizedName: String {
        String(localized: String.LocalizationValue("pain.location.\(rawValue)"))
    }
}

// 用户切换语言后，UI 立即显示新语言 ✅
Text(PainLocation.forehead.localizedName)  // "额头" → "Forehead"
```

### ⚠️ 不支持动态切换的内容

**自定义标签的 displayName**：
- 在首次创建时根据当前语言设置
- 之后不会随系统语言变化而改变
- 这是设计决策，简化架构

## 未来扩展性

### 如需恢复动态语言切换

虽然我们简化了方案，但保留了扩展能力：

1. **`labelKey` 仍然存在**：可用于重新生成 `displayName`
2. **`Localizable.xcstrings` 完整保留**：所有翻译资源仍在
3. **数据迁移脚本**：

```swift
// 如果未来需要支持动态切换，可以运行迁移脚本
func migrateToLocalizedDisplayName() {
    for label in defaultLabels where label.isDefault {
        let locKey = "label.\(label.category).\(label.labelKey)"
        label.displayName = String(localized: String.LocalizationValue(locKey))
    }
}
```
