# 国际化功能修复记录

## 修复日期
2026-02-24

## 背景
在完成国际化实施后，对代码进行全面检查，发现了6个关键问题可能导致功能异常。

## 修复详情

### 1. AnalyticsEngine 统计问题

**问题描述**：
在 `analyzePainLocationFrequency` 方法中，存在两处使用 `displayName` 进行统计的代码。由于 `displayName` 现在返回本地化字符串，这会导致统计结果不稳定。

**受影响文件**：
- `Services/AnalyticsEngine.swift`

**修复方案**：
将 `.map` 改为 `.compactMap`，对无法从 rawValue 构造枚举的情况进行过滤，避免使用 fallback 的 rawValue 字符串。

**修改详情**：
```swift
// 修复前
.map { PainLocationFrequency(
    locationName: PainLocation(rawValue: $0.key)?.displayName ?? $0.key,
    ...
)}

// 修复后
.compactMap { key, count -> PainLocationFrequency? in
    guard let location = PainLocation(rawValue: key) else { return nil }
    return PainLocationFrequency(
        locationName: location.displayName,
        ...
    )
}
```

### 2. CSV 导出本地化问题

**问题描述**：
CSV 导出中使用了枚举的 `rawValue` 和硬编码中文字符串，导致导出的数据没有本地化。

**受影响文件**：
- `Utils/CSVExporter.swift`

**修复方案**：
将所有使用 `rawValue` 的地方改为使用 `localizedName`，使导出的数据使用当前系统语言。

**修改详情**：
- `TriggerCategory.rawValue` → `TriggerCategory.localizedName`
- `PressureTrend.rawValue` → `PressureTrend.localizedName`
- `HealthEventType.rawValue` → `HealthEventType.localizedName`

### 3. PDF 医疗报告本地化问题

**问题描述**：
PDF 报告生成中使用了枚举的 `rawValue`，导致生成的报告没有本地化。

**受影响文件**：
- `Services/MedicalReportGenerator.swift`

**修复方案**：
将使用 `rawValue` 的地方改为使用 `localizedName`。

**修改详情**：
- `BloodType.rawValue` → `BloodType.localizedName`
- `MigraineType.rawValue` → `MigraineType.localizedName`

### 4. 药物类别查找兼容性问题

**问题描述**：
在 `AddMedicationView` 中，从标签的 `subcategory` 查找 `MedicationCategory` 时，只通过 `rawValue` 匹配。由于 `rawValue` 已从中文改为英文键，这会导致无法找到匹配的类别。

**受影响文件**：
- `Views/Medication/AddMedicationView.swift`

**修复方案**：
修改匹配逻辑，同时支持通过 `localizedName` 和 `rawValue` 进行匹配，确保向后兼容。

**修改详情**：
```swift
// 修复前
guard let category = MedicationCategory.allCases.first(where: { 
    $0.rawValue == subcategory 
}) else { return nil }

// 修复后
guard let category = MedicationCategory.allCases.first(where: { 
    $0.localizedName == subcategory || $0.rawValue == subcategory 
}) else { return nil }
```

### 5. 测试数据删除确认多语言支持

**问题描述**：
在 `TestDataView` 中，删除确认只接受中文"确认删除"，不支持英文环境。

**受影响文件**：
- `Views/Testing/TestDataView.swift`

**修复方案**：
添加英文"confirm delete"的支持。

**修改详情**：
```swift
// 修复前
.disabled(clearInputText != "确认删除")

// 修复后
.disabled(clearInputText != "确认删除" && clearInputText != "confirm delete")
```

### 6. 单元测试本地化适配

**问题描述**：
多个单元测试中存在硬编码中文字符串比较，可能在非中文环境下失败。

**受影响文件**：
- `migraine_noteTests/AnalyticsEngineTests.swift`
- `migraine_noteTests/PainLocationTests.swift`

**修复方案**：
- 对于测试数据（通过 TestHelpers 创建的）：保持不变，因为这些是实际的数据值，不是枚举 rawValue
- 对于显示名称测试：改为测试是否非空，而不是测试具体的中文字符串
- 对于统计结果查找：使用更宽松的匹配条件或添加非空断言

**修改详情**：
```swift
// 示例：PainLocationTests.swift
// 修复前
XCTAssertEqual(PainLocation.forehead.displayName, "前额")

// 修复后
XCTAssertFalse(PainLocation.forehead.displayName.isEmpty)
```
