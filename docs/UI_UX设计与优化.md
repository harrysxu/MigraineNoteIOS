## 1. 设计理念演进

### 1.1 第一阶段：v1.0 患者至上设计（2026-02-01）

最初设计文档确立了三大核心理念：

- **患者至上（Patient-First）**：暗黑模式优先（减轻畏光不适）、极简操作（核心记录一键完成）、视觉舒适
- **医疗级严谨（Medical-Grade Accuracy）**：遵循 IHS ICHD-3 与《中国偏头痛诊断与治疗指南》，清晰数据层级
- **情感关怀（Empathetic Experience）**：正向反馈、非侵入式提醒、隐私保护

该版本奠定了数据模型、临床标准和基础设计原则，至今仍是重要依据。

### 1.2 第二阶段：v2.0 医疗极简主义方案（2026-02-03，未采用）

针对「速度至上」提出一套激进重设计：

- **Medical Minimalism**：单色系统（iOS 蓝 + 语义色）、1 步快速记录、扁平设计、去除装饰性动画
- **与情感化设计的差异**：情感化 vs 极简专业、多色 vs 单色、5 步 vs 1 步、呼吸动画 vs 系统默认

该方案强调专业与效率，但未充分考虑用户处于头痛发作时的情绪需求。

### 1.3 第三阶段：情感化设计（2026-02-02，当前方向）

在 v1.0 和 v2.0 之间，团队选择了情感化设计路线，原因包括：

- **用户状态**：偏头痛发作时用户处于痛苦、焦虑状态，需要被支持而非被催促
- **数据支持**：v2.0 的「3 秒记录」虽快，但用户更需要的是降低认知负担和情绪压力
- **差异化**：情感化路线在医疗类 App 中更稀缺，有利于建立情感连接和长期使用习惯

因此，最终采用「痛苦时刻的温柔陪伴」作为主设计方向，并沿用 v1.0 的临床标准与数据模型，同时吸收 v2.0 中 1 步快速记录、智能推荐等思路。

## 2. 当前设计方向

### 2.1 核心理念：「痛苦时刻的温柔陪伴」

在用户最不适的时候，提供平静、支持、易用的体验，而非冷冰冰的录入工具。

### 2.2 参考风格

| 来源 | 借鉴点 |
|------|--------|
| **Apple Health** | 极简的数据层级、克制的配色、清晰的信息架构 |
| **Forest** | 连续天数激励、温暖鼓励、舒缓的视觉节奏 |
| **Calm / Headspace** | 呼吸式动画、友好引导、正向心理学表达 |

### 2.3 设计表达

- **色彩**：治愈蓝绿（#5EC4B6）为主，温暖橙（#F4A261）为辅，低饱和以减少刺激
- **动画**：舒缓呼吸节奏（约 2 秒），避免突兀跳动
- **文案**：鼓励、易懂，减少医学术语带来的焦虑

## 3. 核心设计原则

### 3.1 患者舒适度（Patient Comfort）

- 暗黑模式优先，减少畏光不适
- 柔和色彩、低饱和，降低刺激
- 缓慢、自然的动效，避免刺眼闪烁

### 3.2 医疗准确性（Medical Accuracy）

- 数据结构遵循 IHS ICHD-3 与《中国偏头痛诊断与治疗指南》
- 必填与可选字段区分清晰，便于临床使用
- 术语专业，必要时配通俗说明

### 3.3 情感关怀（Emotional Care）

- 突出「连续无头痛天数」等正向反馈
- 智能诱因推荐，降低记录门槛
- 用温和、非恐吓的方式呈现 MOH 风险提示

### 3.4 可访问性（Accessibility）

- 触摸目标 ≥ 44×44pt
- VoiceOver 全覆盖，重要控件语义清晰
- 支持 Dynamic Type、减弱动画、色盲友好设计

## 4. 视觉优化总结

### 4.1 色彩系统

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 主色 | #6B9ED9 青紫（偏冷） | #5EC4B6 治愈蓝绿 |
| 辅助色 | #BA8FD9 | #4A90E2 柔和蓝、#F4A261 温暖橙 |
| 感受 | 冷静、功能感 | 温暖、信任、平静 |

### 4.2 疼痛强度色阶

| 优化前 | 优化后 |
|--------|--------|
| 高饱和 绿→黄→橙→红 | 低饱和 薄荷绿→柔黄→温橙→珊瑚→柔红 |
| 色彩刺眼，易引发不适 | 饱和度约降 30%，温度感约升 20% |
| 纯色反馈 | 增加情感化文字描述（如「轻微不适」「极度痛苦」） |

### 4.3 卡片样式

| 优化前 | 优化后 |
|--------|--------|
| 直角或小圆角、单一背景 | 圆角增大（约 16pt）、多情感样式 |
| 标准 InfoCard | EmotionalCard 五类：default / elevated / gentle / warning / success |
| 视觉层次较平 | 重要信息用 elevated，提示用 gentle，风险用 warning 等 |

### 4.4 主按钮

| 优化前 | 优化后 |
|--------|--------|
| 扁平、静态 | 渐变背景 + 呼吸光晕动画 |
| 按压反馈较弱 | GentlePressStyle + 触觉反馈 |
| 视觉存在感一般 | 更易被识别为主操作入口 |

### 4.5 记录流程

| 优化前 | 优化后 |
|--------|--------|
| 5 步向导式，步骤固定 | 支持快速记录，可后续补充 |
| 线性疼痛滑块 | 圆形滑块（CircularSlider），360° 拖动 |
| 诱因需逐项翻找 | 智能诱因推荐，基于历史记录 |
| 步骤间跳转频繁 | 单页可折叠分区，减少跳转 |

## 5. 实施的关键组件

### 5.1 EmotionalCard

情感化卡片系统，替代原 InfoCard，支持多种情境：

- `.default`：标准内容块
- `.elevated`：重要信息（浮起感）
- `.gentle`：友好提示、建议
- `.warning`：MOH 等风险
- `.success`：成功、正向反馈

### 5.2 CircularSlider

圆形疼痛评估控件：环形渐变、360° 拖动、大表情反馈、情感化强度文案，支持触觉反馈。

### 5.3 BreathingRecordButton

首页核心记录按钮：呼吸光晕、渐变背景、按压与触觉反馈，降低操作门槛。

### 5.4 Onboarding 引导

首次使用 3 步引导：欢迎与价值、隐私与权限说明、准备就绪与快速开始。

### 5.5 AnimatedNumber

数字变化动画（如连续无头痛天数、统计数据），增强反馈感。

### 5.6 LucideIcons

集成 60+ 医疗健康相关图标，提供 SF Symbols 后备，统一视觉语言。

### 5.7 其他辅助组件

- `DynamicGreeting`：按时段问候（早安/下午/晚上）
- `MiniTrendSparkline`：7 日趋势迷你图
- `EncouragingText`：鼓励性文案
- `AccessibilityHelpers`：无障碍标签与提示

更多实现细节见「设计系统使用指南.md」。

## 6. 临床标准参考

以下内容沿用自 v1.0 设计文档，作为数据与临床表达的规范依据。

### 6.1 国际标准

- **IHS ICHD-3**（International Classification of Headache Disorders, 3rd edition）：头痛分类、发作与先兆定义等
- 症状、先兆、诱因、干预等字段设计与其保持一致

### 6.2 国内指南

- **《中国偏头痛诊断与治疗指南》**：诊断与治疗思路、常用术语、临床路径等
- 术语、提示文案在专业性与可读性之间平衡

### 6.3 数据模型要点

- 必填与可选信息明确区分
- 疼痛部位区域（前额、太阳穴、头顶等）按临床习惯划分
- 支持中医证候相关字段（可选）
- MOH 风险提示与用药频率阈值符合指南建议

## 7. 历史设计参考

### v2.0 医疗极简主义方案（未采用）

v2.0 提出过一套以「Medical Minimalism」为核心的重设计：

- **思路**：单色系统、1 步记录、扁平布局、去掉装饰性动画
- **差异**：强调「速度」和「专业感」，弱化情感与视觉关怀
- **取舍**：团队认为在发作场景下，情感支持与降低焦虑比极致效率更重要，因此选择情感化设计
- **沿用点**：1 步快速记录、AI 智能推荐、可折叠详情等思路被吸收进当前实现

该方案保留为历史参考，未来若需面向更注重专业感的场景（如医生端、科研端），可作为备选方向。

## 8. 设计平衡概览

当前设计在多个维度上保持平衡：

| 维度 | 目标比例 | 说明 |
|------|----------|------|
| 专业性 | 90% | 保持临床标准与数据严谨性 |
| 易用性 | 90% | 智能推荐、快速记录、可折叠详情 |
| 温暖感 | 95% | 色彩、文案、动效共同营造支持感 |
| 极简度 | 85% | 信息密度适中，避免杂乱 |

核心取舍：在保持医疗专业性的前提下，优先强化情感关怀与可访问性，使 App 在用户不适时成为可依赖的陪伴工具，而非冰冷的记录工具。
